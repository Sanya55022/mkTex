         1.  COMMON INFORMATION ABOUT "Создание простых
                        документов LaTex"


     "Создание  простых  документов  LaTex"  является   32битной
консольной  задачей  Windows  и   предназначена   для   Cоздания
технической документации в стандарте LaTex.

     Для отладки и построения использовался компилятор Visual  C
4.0, документы LaTex строились посредством MikTex [1].

     Утилита "Создание  простых  документов  LaTex"  преобразует
текст, набранный в формате RIO [3] , в формат  LaTex-а.  Утилита
может быть использована для быстрого создания

     1) главного документа LaTex-а с типовой преамбулой;

     2) включаемых файлов LaTex-а с
     
          1) перекресными ссылками на термины, разделы;
     
          2) с  гиперссылками  на  внешние  файлы  доступные  по
     протоколу http, ftp, file и т.д.;
     
          3) индексом терминов, содержанием;
     
          4) включаемыми рисунками, ссылками на рисунки;
     
          5) заданными колонтитулами;
     
          6) списками перечисления;
     
          7) двухстрочными и многострочными пояснениями;
     
          8) тестом с математическими формулами;
     
          9) примечаниями.

     Таким образом простой набор  из  двух  дюжин  команд  легко
позволяет создавать техничекие  документы,  достаточно  высокого
качества. пользоваться LaTex-ом.

           2.  HOW TO BUILD AND RUN "Создание простых
                        документов LaTex"


     Для построения утилиты наберите (после  того  как  согласно
здоровью и полученному образованию правильно зададите  параметры
для make файла, лежащие в файле parms.inc):

mk mkTex


     Для построения документации необходима утилита RIO:

cd ./docs
rio <mkTex.rio >mkTex.txt


     Для получения подсказки наберите:

 usage LaTex generator version of 1.21.0:
 G:\agp\96\mkTex\bld\mkTex.exe [preamble[={1|2|10|12}]][;class=xxx][;rsl][;skipAppendix][;dbg][;letter;doxygen]  <input >output
 	where
 preamble    - create preamble
 preamble=1  - create preamble and initial for cp866(emtex)
 preamble=2  - create preamble and initial for cp1251(miktex) and add
      '\usepackage[russian]{babel}'
 preamble=10 - create preamble for pdflatex(as doxygen makes)
 preamble=12 - as 10 and add babel package
 class=xxx   - xxx isin {article,report,book,etc}
 rslenv      - to use package for rsl
 doxygen     - to use crossreferens. additinal command: .x ?metka word
 tsize=N     - size of normal text. O<=N<10 (tiny, small, normalsize:4, large, Large:6, etc)
 vsize=N     - size of verbatim text
 there is the special case of rio comments:
 .;# - swith on the bibliography instead the items (.(, .+, .))
 .;\c ref - +> .\cite{ref} - make the reference to bibliography
 .;\bla bla bla - just copy \bla bla bla to a tex text


     - В режиме  doxygen  включается  генерация  гиперссылок  на
термины, заносимые в индекс.

     - Параметры tsize и vsize задают размеры нормального шрифта
и  шрифта  для  текста,  выводимого   без   изменения.   Таблица
соответствия размеров

0      tiny
1      scriptsize
2      footnotesize
3      small
4      normalsize
5      large
6      Large
7      LARGE
8      huge
9      Huge


     Для построения примера ./add/bibtex/bibTest.pdf наберите
cd ./add/bibtex
bibTest.cmd

         3.  DESCRIPTION OF "Создание простых документов
                             LaTex"


     "Создание простых документов LaTex" заменяет (ну почти все)
команды и  макромепеменные  rio  на  соответствующие  команды  и
переменные LaTex. Можно также  включать  команды  LaTex  в  виде
коментария  в   текст   rio,   используя   символ   бакслеша   -
.;\команда_латеха.    Библиография    включается     посредством
специального коментария (с точки зрения rio) - .;#.
     
     
     3.1.  Специальные символы LaTex-а

     Некоторые символы, а именно - \, {, }, %, &,  ~,  $,  ^,  #
являются специальными символами LaTex. Чтобы они смогли  попасть
в документ их надо  предварять  символом  '\'.  Утилита  заменит
цепочку \x на правильныю цепочку для вывода символа x в документ
LaTex-а.

     В противном случае, например, вхождение символа $ переведет
LaTex из текстового режима в математический. Цепочка  $\pi_{12}$
приведет к появлению в выходном документе LaTex-а символа  числа
ПИ с нижним индексом 12.

     3.3.  Внутренние переменные RIO


     
@_ - обозначение документа;
     
@Y - две последних цифры текущего года (из системной даты);
     
@M - номер текущего месяца (из системной даты);
     
@D - номер текущего дня месяца (из системной даты);
     
@P - текущий номер страницы;
     
@L - текущее значение левого поля;
     
@R - текущее значение правого поля;
     
@C - текущий номер главы;
     
@1 - текущий номер раздела;
     
@2 - текущий номер подраздела;
     
@3 - текущий номер пункта;
     
@" - текущий номер нумеруемого приложения.

     3.5.  Расширение набора команд RIO

     В командах рубрики .X, где X число от 1 до  9,  допускается
ставить метку. Метка начинается символом +. Например,

  .X +metka Название рубрики

далее, для ссылки на раздел эта метка используется в команде .;

               
.;#          -      переключиться на оформление библиографии
               
.;\          - команда Tex. Текст, следующий  за  цепочкой  '.;'
               копируется в выходной поток без изменение;
               
.;-          - не менять текст. Строчки, находящиеся между двумя
               такими командами попадут  в  выходной  поток  без
               изменений;
               
.9           - рубрика  изменяемого  уровня.  В  выходной  поток
               попадает  текст  .x  где  x  -  число.  Начальное
               значение равно 4;
               
.;u          - увеличить уровень рубрики .9;
               
.;d          - уменьшить уровень рубрики .9;
               
.;c metka    - ссылка на  литературу.  Компилируется  в  команду
               LaTex-а \cite{metka} ;
               
.;l metka    - установка метки в произвольном месте(в режиме do-
               xygen).   Компилируется   в    команду    LaTex-а
               \label{metka} ;
               
.;p metka    - - гиперссылка на страницу (в режиме doxygen),  на
               которой находится метка  metka.  Компилируется  в
               команду LaTex-а \pageref{metka} ;
               
.;+metka     - - гиперссылка на раздел  (в  режиме  doxygen),  в
               котором появилась метка  metka.  Компилируется  в
               команду LaTex-a \ref{metka}. Метка  metka  должна
               появиться или в команде установки метки .;l или в
               команде рубрики.
               
.x ?m слово  - (в   режиме   doxygen)   гиперссылка   на   слово
               \hyperlink{м}{слово}\index{слово}. цепочка  слово
               выделяется  другим  цветом,  при  клике  на  нее,
               происходит переход на метку m ;
               
.x +m слово  - установка метки  (в  режиме  doxygen)  на  слово,
               заносимое                в                индекс.
               \hypertarget{m}{\index{слово}}\label{m} .

                            4.  USAGE


     В   каталоге   ./add/bibtex   находится   командный    файл
bibTest.cmd для построения pdf- документа с  гиперссылками  и  c
использованием базы данных библиографии. Его содержимое

dot.exe      -Tps  mkDoc.dot  -o.mkDoc.ps
epstopdf.exe .mkDoc.ps --outfile=mkDoc.pdf
mkTex.exe    class=article;letter;preamble=12 <bibTest.txt > .bibTest.tex
pdflatex.exe -interaction=batchmode  -terminal=oem  .bibTest.tex
makeindex    2>.makeindex.log .bibTest.idx -o .bibTest.ind
bibtex       -include-directory=./bibDB  .bibTest   1>.bibTex.log
pdflatex.exe -interaction=batchmode  -terminal=oem  .bibTest.tex
pdflatex.exe -interaction=batchmode  -terminal=oem  .bibTest.tex


     Файл bibTest.txt  представляет  собой  пример  документа  с
гиперссылками.

     База данных расположена в каталоге ./add/bibtex/bibDB.

     Построенный документ - bibTest.pdf.
